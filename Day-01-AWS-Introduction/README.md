# ☁ Day 01 - AWS Introduction

## Objective

To understand the fundamentals of Cloud Computing and Amazon Web Services (AWS), its advantages, and the core concepts behind cloud infrastructure.

---

# What is Cloud Computing?

Cloud Computing is the on-demand delivery of computing services such as servers, storage, databases, networking, and software over the internet with pay-as-you-go pricing.

Instead of purchasing and maintaining physical servers, users can access computing resources from cloud providers like AWS.

---

# What is AWS?

Amazon Web Services (AWS) is a cloud computing platform provided by Amazon that offers more than 200 cloud services for computing, storage, networking, databases, analytics, security, and machine learning.

AWS is one of the world's leading cloud service providers and is used by startups, enterprises, and government organizations.

---

# Why Cloud Computing?

## Traditional Infrastructure Challenges

* High initial hardware cost
* Maintenance and upgrades
* Limited scalability
* Physical infrastructure management
* Downtime risks

## Cloud Computing Advantages

* Pay only for what you use
* High availability
* Global access
* Easy scalability
* Better security
* Faster deployment

---

# Types of Cloud Computing

## 1. Public Cloud

Resources are owned and managed by third-party cloud providers and delivered over the internet.

Examples:

* AWS
* Microsoft Azure
* Google Cloud Platform (GCP)

---

## 2. Private Cloud

Cloud infrastructure is used exclusively by a single organization.

Examples:

* VMware
* OpenStack

---

## 3. Hybrid Cloud

Combination of both Public Cloud and Private Cloud.

Example:
An organization stores sensitive data on-premises and uses AWS for application hosting.

---

# Cloud Service Models

## Infrastructure as a Service (IaaS)

Provides virtual machines, storage, and networking.

Examples:

* Amazon EC2
* Amazon EBS

---

## Platform as a Service (PaaS)

Provides a platform for developing and deploying applications.

Examples:

* AWS Elastic Beanstalk
* AWS Lambda

---

## Software as a Service (SaaS)

Provides software applications over the internet.

Examples:

* Gmail
* Microsoft 365
* Salesforce

---

# AWS Global Infrastructure

AWS infrastructure consists of:

## Regions

A geographical location where AWS has multiple data centers.

Examples:

* Mumbai (ap-south-1)
* Singapore (ap-southeast-1)
* US East (N. Virginia)

---

## Availability Zones (AZ)

One or more discrete data centers within a region.

Benefits:

* High availability
* Fault tolerance
* Disaster recovery

---

## Edge Locations

Used by Amazon CloudFront to deliver content with low latency.
